# 画面一覧 — デルカク✓

MVP機能（1〜8）と対応させた画面構成。全14画面（うちモーダル3つ）。

| No | 画面名 | 概要 | 対応するMVP機能 | 種別 |
|---|---|---|---|---|
| S01 | オンボーディング | 初回起動時のアプリ説明（3スライド程度）と通知許可リクエスト | 通知機能の前提 | フルスクリーン |
| S02 | ホーム | 今日／次回の予定の有無、未完了チェックの有無、主要機能への入口 | 全体のハブ | フルスクリーン |
| S03 | チェック実行 | 通知タップ後に開く、大きなチェックボックスのリスト | 6. チェック画面 | フルスクリーン |
| S04 | 完了 | 全項目チェック後に表示する完了演出 | 7. 完了条件 | フルスクリーン |
| S05 | スヌーズ選択 | 5分後／10分後／15分後を選ぶ | 8. スヌーズ | モーダル |
| S06 | 項目一覧 | 登録済みチェック項目の一覧（編集・削除） | 1. チェック項目管理 | フルスクリーン |
| S07 | 項目追加・編集 | 項目名・アイコン選択 | 1. チェック項目管理 | モーダル |
| S08 | セット一覧 | 登録済みセットの一覧（編集・削除） | 2. セット機能 | フルスクリーン |
| S09 | セット追加・編集 | セット名＋含める項目を複数選択 | 2. セット機能 | フルスクリーン |
| S10 | カレンダー | 月表示カレンダー、日付ごとの予定（セット／項目）一覧 | 3. カレンダー登録 | フルスクリーン |
| S11 | 予定追加・編集 | 日付／曜日指定、セット・個別項目の選択、通知時刻設定 | 3. カレンダー登録、5. 通知機能 | フルスクリーン |
| S12 | 設定 | 通知許可状態、文字サイズ、デフォルトスヌーズ時間、データ初期化 | 5. 通知機能（全体設定） | フルスクリーン |
| S13 | 重複統合確認（任意表示） | 複数セット登録時、統合後リストをプレビュー表示 | 4. 重複排除 | モーダル（必要に応じて） |

> 4.重複排除はユーザー操作を必要とするUIではなく、S03（チェック実行画面）の生成時に内部ロジックで自動統合する。S13は確認のための補助画面であり、MVPでは省略可（[05-mvp-tasks.md](05-mvp-tasks.md)で優先度を「任意」として扱う）。

## 画面ごとの主要要素（抜粋）

### S02 ホーム
- 「今日の予定」カード（予定がある場合はセット名・時刻、ない場合は「予定なし」）
- 「今すぐチェックを始める」大ボタン（予定がなくても任意実行可能）
- カレンダー／項目管理／セット管理／設定への導線（アイコン＋ラベルの大きいタップ領域）

### S03 チェック実行
- チェック項目を縦に大きく並べる（1項目＝1行、タップで✓）
- 全項目表示後、自動で完了画面へ遷移
- 「あとで」ボタン → S05スヌーズ選択へ

### S04 完了
- 「確認完了！」のメッセージとアニメーション
- ホームへ戻るボタンのみ（迷わせない）

### S12 設定
- 通知許可状態の表示と再リクエスト導線
- 文字サイズ（標準／大／特大）
- データの書き出し・初期化（バックアップが取れない不安への対応、[07-improvements.md](07-improvements.md)参照）
